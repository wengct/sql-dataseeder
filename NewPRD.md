目前 `sql-dataseeder` 僅支援根據欄位型別自動產生假資料，無法直接使用資料庫中現有資料作為輸出來源。

為了提升工具的靈活性與真實性，建議增加一個  **資料來源選擇功能** ，允許使用者在產生 SQL INSERT 指令時，能夠從下列來源中選擇：

1. **假資料生成（現有功能）**
2. **使用當前資料庫實際資料**

* * * * *

## 🎯 功能需求細節

### 1\. 於產生 SQL 指令時加入「資料來源選擇」設定

新增一個「Generate Existed Insert Scripts」：

- 使用假資料
- 使用當前資料庫的資料（可選擇筆數或條件）

### 2\. 支援從資料庫讀取資料表內容

若選擇「從資料庫讀取資料」：

- 從目前連線的資料庫擷取目標資料表內容
- 可支援自訂筆數（例如：前 N 筆）
- 可支援條件篩選（例如：WHERE、ORDER BY）
- 擷取的資料需自動轉成 INSERT 語法

### 3\. 產生的 SQL 結果需符合目前工具格式

- 自動複製到剪貼簿
- SQL 語法需正確處理字串、日期、NULL、特殊字元等
- 維持現在的 SQL 排版風格
